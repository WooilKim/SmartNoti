# SmartNoti Device Functional Test Checklist

## 목적
실기기에서 SmartNoti가 더미 데이터가 아닌 실제 알림 데이터를 읽고 분류하는지 검증한다.

## 사전 조건
- 최신 `main` 브랜치 빌드 설치
- 앱 첫 실행 전 기존 앱 데이터 삭제 권장
- 테스트 기기에서 다른 앱 알림이 정상적으로 오는 상태

## 1. 첫 설치 / 온보딩
- [ ] 앱 실행 시 Home이 아니라 권한 온보딩이 표시된다.
- [ ] 필수 권한이 충족되기 전에는 메인 앱으로 진행할 수 없다.

## 2. 알림 접근 권한
- [ ] 온보딩의 알림 접근 설정 버튼이 시스템 설정으로 이동한다.
- [ ] 시스템 설정에서 SmartNoti 알림 접근을 켤 수 있다.
- [ ] 앱으로 돌아오면 권한 상태가 즉시 갱신된다.

## 3. Android 13+ 알림 표시 권한
- [ ] Android 13 이상에서는 알림 표시 권한 요청이 보인다.
- [ ] 권한 허용 후 상태가 갱신된다.
- [ ] Android 12 이하에서는 이 단계가 나타나지 않는다.

## 4. 온보딩 완료
- [ ] 필수 권한 충족 후 시작하기 버튼이 활성화되거나 자동으로 완료된다.
- [ ] 완료 후 Home 화면으로 이동한다.

## 5. 실데이터 확인
- [ ] 초기 상태에서 실제 캡처 알림이 없으면 빈 상태가 보인다.
- [ ] 더미 예시 알림이 표시되지 않는다.

## 6. 실제 알림 발생 테스트
다음 중 하나 이상으로 실제 알림을 발생시킨다.
- 카카오톡 / 문자 / 이메일 / Slack 알림 수신
- 인증번호 문자 수신
- 쇼핑앱 / 배달앱 푸시 수신
- 같은 앱에서 반복 알림 발생

검증:
- [ ] Home 화면에 실제 알림이 나타난다.
- [ ] Priority / Digest / Silent 분류 수가 실제 데이터 기준으로 바뀐다.
- [ ] 에뮬레이터에서 반복/프로모션/중요 시나리오를 재현해야 할 경우 `docs/real-notification-verification-with-testnotifier.md` 절차를 사용한다.

## 7. Priority 화면
- [ ] 중요 알림만 표시된다.
- [ ] 알림 상세 화면으로 이동할 수 있다.

## 8. Digest 화면
- [ ] 덜 중요한 알림이 앱별 또는 묶음 형태로 표시된다.
- [ ] 그룹 개수와 미리보기가 실제 알림 기준으로 보인다.

## 9. Detail 화면
- [ ] 선택한 실제 알림의 앱명 / 발신자 / 본문 / 상태 / reason tags가 표시된다.
- [ ] 다음 액션이 동작한다.
  - [ ] 중요로 고정
  - [ ] Digest로 보내기
  - [ ] 조용히 처리

## 10. 규칙 학습 반영
- [ ] 상세 화면에서 학습 액션 수행 후 규칙이 저장된다.
- [ ] 비슷한 알림 재수신 시 같은 방향으로 분류된다.

## 11. 앱 재실행
- [ ] 앱 재실행 시 온보딩이 다시 뜨지 않는다.
- [ ] 저장된 실제 알림이 유지된다.

## 성공 기준
- [ ] 권한 온보딩이 동작한다.
- [ ] 권한 완료 전 메인 진입이 막힌다.
- [ ] 권한 완료 후 Home 진입이 가능하다.
- [ ] 더미 데이터 없이 실제 알림만 보인다.
- [ ] 실제 알림이 Home / Priority / Digest / Detail에 반영된다.
- [ ] 학습 액션과 재분류가 동작한다.

## 실패 시 우선 점검
### 알림이 하나도 안 들어옴
- Notification Access가 실제로 켜져 있는지 확인
- 제조사 배터리/백그라운드 제한 확인
- 테스트 알림이 실제로 발생했는지 확인

### 앱은 열리지만 화면이 비어 있음
- 테스트 알림이 아직 발생하지 않았는지 확인
- 기존 앱 데이터/재설치 영향 확인
- 오래된 APK가 아닌 최신 빌드인지 확인

### 온보딩이 반복됨
- 권한 상태가 실제로 충족되었는지 확인
- 앱 데이터 저장이 막히지 않았는지 확인

### 알림센터에서 원본 알림이 계속 보임
- 현재 버전은 원본 앱 알림을 읽고 분류하지만 기본적으로 완전한 시스템 수준 차단/대체까지 보장하지는 않는다.
- 이 동작은 Android 정책과 NotificationListenerService 제약을 별도로 검토해야 한다.
